# ChaeOn — 커뮤니케이션 감정·공격성 분석 AI

## 1. 프로젝트 개요

**ChaeOn**은 팀 채팅/메신저 대화 텍스트를 입력받아 **감정 상태**와 **공격성 여부**를
분석하고, 팀 커뮤니케이션의 **위험 신호를 조기에 탐지**해 보여 주는 AI 프로젝트입니다.

개별 메시지의 감정·공격성 분석에 그치지 않고, 이를 시간·발신자 단위로 집계하여
**소통 편중도**와 **팀 분위기 변화**까지 추적합니다. 최종적으로 팀 프로젝트 과정에서
발생할 수 있는 갈등과 소통 문제를 사람이 인지하기 전에 시각적으로 드러내는 것을
목표로 합니다.

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | ChaeOn |
| 한 줄 정의 | 커뮤니케이션 감정·공격성 분석 AI |
| 입력 | 팀 채팅/메신저 텍스트 (타임스탬프, 발신자 정보 포함) |
| 출력 | 감정 수치, 공격성 판정/확률, 위험 신호 리포트, 분위기 변화 그래프 |

## 2. 해결하려는 문제

- 팀 프로젝트 환경의 메신저 기반 커뮤니케이션은 **텍스트 중심 소통**이라
  감정 전달에 한계가 있고, **공격적인 표현이 누적**되기 쉽습니다.
- ChaeOn은 채팅 내 **감정 변화와 공격성 표현을 분석**하여
  **커뮤니케이션 위험 신호를 조기에 탐지**하는 것을 목표로 합니다.
- 특정 시점의 부정 감정 증가, 공격적 표현 빈도 증가, 대화 분위기 변화 등을
  **시각적으로 확인**할 수 있도록 지원합니다.
- 이를 통해 팀 프로젝트 과정에서 발생할 수 있는 **갈등 상황과 소통 문제를
  더 빠르게 인지하고 대응**할 수 있도록 돕습니다.

## 3. 주요 기능

ChaeOn은 5개 기능으로 구성됩니다. 기능 1·2는 **AI 모델**이며, 기능 3·4·5는
모델 결과 또는 통계를 **집계·시각화·해석**하는 분석 계층입니다.

> **이번 구현 범위**: 기능 1~4만 포함합니다. 기능 5(팀 분위기 변화 추적)는
> 이번 구현에서 제외되며, 향후 **확장 가능성**으로만 명시합니다.
>
> **확정 상태**: 기능 3~5(분석 계층)는 아직 확정된 사항이 아닙니다. 아래
> 내용은 현재 구상안이며, **기획 구체화 이후 세부 설계를 추가로 확정**합니다.

### 3.3. 결과 시각화 / 위험 신호 분석 (분석 계층 · 미확정)

> 기능 1·2 모델 결과를 기반으로 분석 리포트를 제공합니다.
> 별도 AI 모델이 아니라, 모델 결과를 **집계·시각화·해석**하는 기능입니다.
>
> ⚠️ **미확정**: 아래 항목은 구상안 예시이며, 기획 구체화 이후 확정됩니다.

- **특정 팀원의 부정 감정 비율 증가**
  - 예: 팀원 A의 부정 감정이 최근 1시간 내 평소 대비 3.5배 증가 → 위험 신호
- **공격적 표현 빈도 증가**
  - 예: 팀원 A의 공격적 표현 수가 최근 1시간 내 평소 대비 5배 증가 → 위험 신호
- **특정 시간대 갈등 집중**
  - 예: 특정 요일/시간대의 전체 메시지 대비 공격성 메시지 비율 계산
- **팀 전체 감정 흐름 변화**
  - 예: 자료 제출 직전 시점에 부정 비율이 증가하는 패턴 탐지

### 3.4. 소통 편중도 분석 (분석 계층 · 미확정)

> ⚠️ **미확정**: 아래 내용은 구상안이며, 기획 구체화 이후 확정됩니다.

- **입력**: 메신저 대화 텍스트 (타임스탬프, 발신자 정보 포함)
- **출력**: 팀원별 발화 비중, 질문 빈도, 평균 응답 시간(Latency), 특정 인원 소외 지표
- **데이터셋**: 프로젝트 참여 팀원들의 실제 대화 로그 (익명화 처리 필수)
- **분석 방법**:
  - *Feature Extraction*: 발신자별 메시지 수 카운트, 문장 끝 물음표(?) 기반 질문 추출
  - *Statistical Analysis*: 팀원 간 메시지 비중 편차 계산, 메시지 발생 시각 차이로
    응답 속도 산출
- **참고**: 학습 모델이 아닌 **정량적 통계 데이터**를 활용한 지표 산출

### 3.5. 팀 분위기 변화 추적 (분석 계층 · 확장 가능성 — 이번 구현 제외)

> 🔭 **확장 가능성**: 기능 5는 이번 구현 범위에 포함되지 않습니다. 아래
> 내용은 향후 확장 시 참고할 구상안이며, 기획 구체화 이후 확정됩니다.

- **입력**: 기능 1·2에서 산출된 감정/공격성 수치 + 타임스탬프
- **출력**: 시간 흐름에 따른 팀 전체/개인별 감정 곡선 및 공격성 변동 추이 그래프
- **데이터셋**: 기능 1·2의 분석 결과 데이터셋
- **분석 방법**:
  - *Time-Series Smoothing*: 이동평균(Moving Average) 등으로 시간대별 감정 흐름을
    시각적으로 매끄럽게 변환
  - *Anomaly Detection*: 평소 팀 분위기(Baseline) 대비 부정 수치가 급격히 튀는
    구간(Outlier)을 자동 탐지
- **참고**: 프로젝트 진행 단계(초기-중기-마감)별 분위기 변화를 리포트로 제공

## 5. 시스템 흐름

```
메신저 대화 텍스트 (타임스탬프 + 발신자)
        │
        ├─▶ [기능 1] 감정 분류 모델 ───┐
        │                              ├─▶ 감정/공격성 수치
        ├─▶ [기능 2] 공격성 탐지 모델 ─┘
        │
        ├─▶ [기능 3] 위험 신호 분석  ── 모델 결과 집계·해석

   ┄┄▶ [기능 5] 팀 분위기 변화 추적 ── 확장 가능성 (이번 구현 제외)
```

## 6. 적용 범위와 제약

- 기능 1·2만 학습 기반 AI 모델이며, 기능 3·4는 **모델 출력 또는 통계의
  집계·시각화·해석** 계층입니다.
- **이번 구현 범위는 기능 1~4**입니다. 기능 5(팀 분위기 변화 추적)는
  확장 가능성으로만 명시하며 이번 범위에서 제외합니다.
- 기능 3~5(분석 계층)는 아직 확정된 사항이 아니며, 기획 구체화 이후
  세부 설계를 추가로 확정합니다.
- 분석 대상이 되는 팀 대화 로그는 **반드시 익명화 처리** 후 사용합니다.
- 위험 신호 판정 기준(예: "1시간 내 평소 대비 N배")은 운영 중 보정 가능한
  설정값으로 다룹니다.
