# 2. 데이터 폴더 구조

## 왜 폴더를 나누나

데이터는 한 번에 완성되지 않습니다. 원본 → 정리 → 청소 → 숫자 변환 순으로
**여러 번 가공**됩니다. 각 가공 단계의 결과를 폴더로 분리해 두면:

- 어디까지 처리했는지 한눈에 보임
- 앞 단계를 다시 안 해도 됨 (모델만 바꿔 실험할 때 유용)
- 팀원끼리 "그 파일 어디 있어?"가 사라짐

## 4단계 표준 구조

```
data/
├── raw/         ← 원천 데이터 (다운로드 원본, 절대 수정 안 함)
├── labeled/     ← 라벨링/라벨 정리 완료 데이터
├── processed/   ← 전처리(청소) 완료 데이터
└── features/    ← 벡터화(숫자 변환) 결과
```

### raw/ — 원천 데이터

AIHub, K-HATERS 등에서 **다운로드한 원본 그대로**. 절대 직접 고치지 않습니다.
원본을 보존해야 나중에 문제가 생겨도 처음부터 다시 할 수 있습니다.

> ChaeOn에서: AIHub 감성 대화 말뭉치, 텍스트 윤리검증 데이터의 압축 파일과
> 개별 JSON들, `merge_data.py`가 만드는 병합 JSON이 여기 들어갑니다.
> 용량이 크고 외부 데이터라 git에는 올리지 않습니다 (`.gitignore` 처리).

### labeled/ — 라벨링 완료 데이터

**문장과 정답(라벨)이 짝지어진** 데이터. "이 문장 = 공격적", "저 문장 = 긍정".

- 라벨이 **없는** 데이터를 모았다면 → 여기서 사람이 직접 라벨을 답니다.
- 라벨이 **이미 있는** 데이터라면 → 데이터셋마다 제각각인 포맷을
  `(문장, 라벨)` 형태의 **통일된 표**로 추출·정리합니다.

> ChaeOn에서: 우리가 쓰는 데이터셋(AIHub·K-HATERS·K-MHaS)은 **이미 라벨이
> 붙어 있습니다.** 그래서 "사람이 라벨링하는 작업"은 없고, 데이터셋별로
> 다른 컬럼 구조를 `text, label` 두 컬럼짜리 CSV로 통일하는 용도로 씁니다.

### processed/ — 전처리 완료 데이터

라벨 데이터의 **텍스트를 청소**한 결과. 특수문자 제거, 중복 공백 정리,
형태소 분리(토큰화) 등 → 자세한 건 [03 문서](03-vectorization.md).

> ChaeOn에서: 채팅 특유의 이모티콘·자음("ㅋㅋ")·오타 등을 다듬어
> 모델이 학습하기 좋은 깨끗한 문장으로 만듭니다.

### features/ — 벡터화 결과

청소된 문장을 **숫자(벡터)로 변환**한 결과. 모델은 글자를 못 읽고 숫자만
이해하기 때문에 반드시 거치는 단계입니다 → [03 문서](03-vectorization.md).

> ChaeOn에서: `CountVectorizer`·`TF-IDF`로 만든 숫자 행렬과, 그 변환 규칙을
> 담은 vectorizer 파일이 들어갑니다.

## ChaeOn의 현재 선택: 3폴더 vs 4폴더

현재 저장소 `data/` 폴더는 **3개**입니다 — `raw / processed / features`
(`labeled/`가 없음).

이건 틀린 게 아닙니다. **`labeled/`는 "라벨을 직접 만드는 단계"가 있을 때
의미가 큰 폴더**인데, ChaeOn이 쓰는 데이터셋은 전부 제공처에서 라벨이
완성된 상태로 옵니다. 즉 직접 라벨링할 일이 없습니다.

| 선택 | 언제 적합 | ChaeOn 적용 |
| --- | --- | --- |
| 3폴더 (raw/processed/features) | 라벨이 이미 있는 공개 데이터만 쓸 때 | `(text,label)` 통합 표는 `processed/`에 둠 |
| 4폴더 (labeled 추가) | 라벨 통합·정리 단계를 명시하고 싶을 때 | `labeled/`를 "라벨 추출·통합" 단계로 재정의 |

**둘 다 가능합니다.** 팀이 정하면 됩니다. 단, 4폴더로 간다면 `labeled/`를
빈 폴더로 두지 말고 "데이터셋별 통합 CSV를 두는 곳"으로 역할을 분명히 정하세요.

## 규칙 한 가지

**raw/ 안의 파일은 절대 수정하지 않습니다.** 가공이 필요하면 항상
"원본을 읽어서 → 다음 폴더에 새로 저장"하는 식으로 진행하세요.
원본이 살아 있어야 실수해도 되돌릴 수 있습니다.
