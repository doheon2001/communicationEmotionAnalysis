# 6. 용어 사전

ChaeOn 문서·코드에서 나오는 ML 용어를 한 줄로 정리했습니다.
더 자세한 설명은 괄호 안 문서를 보세요.

## 데이터 관련

- **데이터셋 (dataset)**: 모델을 학습/평가하는 데 쓰는 데이터 묶음.
- **라벨 (label)**: 문장에 붙은 정답. 예: "공격적", "긍정". (01)
- **멀티레이블 (multi-label)**: 한 문장에 라벨이 여러 개 동시에 붙는 것.
  K-MHaS가 8개 클래스 멀티레이블. (spec.md)
- **전처리 (preprocessing)**: 텍스트를 청소·정리하는 단계. (03)
- **토큰화 (tokenization)**: 문장을 단어/형태소 단위로 쪼개는 것. (03)
- **익명화**: 누가 한 말인지 알 수 없게 개인정보를 지우는 것. 팀 대화 로그
  사용 시 필수. (spec.md)
- **데이터 누수 (data leakage)**: 시험 데이터가 학습에 새어 들어가 점수가
  부풀려지는 실수. (05)
- **불균형 데이터 (imbalanced data)**: 한쪽 라벨이 다른 쪽보다 훨씬 많은
  데이터. 정확도만 보면 속기 쉬움. (05)

## 벡터화 관련

- **벡터화 (vectorization)**: 글자를 모델이 읽는 숫자로 바꾸는 것. (03)
- **벡터 (vector)**: 숫자들의 목록. 벡터화의 결과물. (03)
- **BoW (Bag of Words)**: 단어 순서를 무시하고 개수만 세는 방식. (03)
- **CountVectorizer**: 단어 출현 횟수를 세어 벡터로 만드는 도구. (03)
- **TF-IDF**: 흔한 단어는 약하게, 희귀한 단어는 강하게 가중치를 주는 벡터화
  방식. (03)
- **vectorizer**: 벡터화 규칙을 담은 객체. 학습·예측에 같은 것을 써야 함. (03)

## 모델 관련

- **모델 (model)**: 데이터에서 학습한 패턴 덩어리. 입력을 받아 결과를 냄. (01)
- **분류 (classification)**: 입력을 정해진 칸 중 하나로 나누는 작업. (04)
- **학습 (training)**: 정답 있는 데이터로 모델이 패턴을 익히는 과정. (01)
- **예측 (inference)**: 학습된 모델이 새 입력에 결과를 내는 것. (01)
- **로지스틱 회귀 (Logistic Regression)**: 단어별 점수를 더해 확률로 바꾸는
  단순·빠른 분류 모델. (04)
- **나이브 베이즈 (Naive Bayes)**: 확률 계산 기반 분류 모델. 스팸 필터의
  고전. (04)
- **랜덤 포레스트 (Random Forest)**: 결정 트리 여러 개의 다수결 모델. (04)
- **결정 트리 (decision tree)**: "예/아니오" 질문을 이어 가며 분류하는 모델.
  스무고개와 비슷. (04)
- **predict**: 결과를 라벨로 딱 잘라 주는 모델 메서드. 예: "부정". (04)
- **predict_proba**: 결과를 확률로 주는 모델 메서드. 예: 부정 0.87. ChaeOn이
  주로 쓰는 출력. (04)
- **과적합 (overfitting)**: 학습 데이터만 외워서 새 데이터에 약해진 상태. (05)
- **베이스라인 (baseline)**: 비교 기준이 되는 가장 기본적인 모델/성능. (04)

## 평가 관련

- **학습 데이터 (train set)**: 모델이 공부하는 데 쓰는 데이터. (05)
- **시험 데이터 (test set)**: 모델 채점에 쓰는, 학습에 안 쓴 데이터. (05)
- **혼동 행렬 (confusion matrix)**: 맞춤/틀림을 네 칸으로 정리한 표. (05)
- **정확도 (accuracy)**: 전체 중 맞춘 비율. 불균형 데이터에선 주의. (05)
- **정밀도 (precision)**: 공격적이라 예측한 것 중 진짜인 비율. 낮으면 오탐
  많음. (05)
- **재현율 (recall)**: 실제 공격적인 것 중 잡아낸 비율. 낮으면 놓침 많음. (05)
- **F1 점수**: 정밀도와 재현율의 균형 점수. 모델 비교의 주 지표. (05)
- **오탐 (False Positive, FP)**: 아닌 것을 맞다고 한 오류. (05)
- **놓침 (False Negative, FN)**: 맞는 것을 아니라고 한 오류. (05)

## 도구

- **scikit-learn (sklearn)**: ChaeOn이 쓰는 파이썬 ML 라이브러리. 위 모델·
  벡터화·평가 지표가 모두 들어 있음.
- **KoNLPy**: 한국어 형태소 분석(토큰화) 라이브러리. (03)
- **파이프라인 (pipeline)**: 데이터 수집→예측까지 이어지는 처리 단계 전체. (01)
