# 1. 머신러닝 전체 흐름

## 머신러닝이 뭔가요

머신러닝(ML)은 **"규칙을 사람이 직접 코딩하지 않고, 데이터에서 규칙을 자동으로
찾게 하는 것"**입니다.

예를 들어 "공격적인 문장을 골라내라"는 작업을 생각해 봅시다.

- **규칙 기반 (옛날 방식)**: 욕설 단어 목록을 사람이 만들어서 "이 단어가 있으면
  공격적" 이라고 코딩 → 새로운 표현이 나오면 계속 손으로 추가해야 함.
- **머신러닝 방식**: "공격적/비공격적" 라벨이 붙은 문장 19만 건을 모델에게
  보여 주면, 모델이 스스로 "어떤 단어 조합이 공격적인지" 패턴을 학습함.

ChaeOn은 머신러닝 방식을 씁니다. 사람이 욕설 사전을 만드는 대신,
**라벨이 달린 공개 데이터셋**으로 모델을 학습시킵니다.

## 핵심 용어 두 개

- **학습(training)**: 정답(라벨)이 있는 데이터를 모델에게 보여 주며 패턴을
  익히게 하는 과정. "공부하는 단계".
- **예측(inference)**: 학습이 끝난 모델에게 **정답을 모르는 새 문장**을 주고
  결과를 받는 과정. "시험 보는 단계".

ChaeOn에서: 공개 데이터셋으로 **학습** → 실제 팀 채팅에 대해 **예측**.

## ChaeOn의 ML 파이프라인 6단계

```
 ① 데이터 수집      raw/         AIHub·K-HATERS·K-MHaS 원본 다운로드
        │
 ② 라벨 정리        labeled/     (문장, 정답라벨) 통합 표로 추출
        │
 ③ 전처리          processed/   불필요한 문자 제거·정규화·토큰화
        │
 ④ 벡터화          features/    글자 → 숫자(벡터)로 변환 (TF-IDF 등)
        │
 ⑤ 모델 학습        models/      로지스틱 회귀 등으로 패턴 학습
        │
 ⑥ 평가 & 예측                  정확도 측정 → 실제 채팅에 적용
```

| 단계 | 하는 일 | ChaeOn에서 |
| --- | --- | --- |
| ① 수집 | 원천 데이터 모으기 | AIHub 감성 대화, 텍스트 윤리검증, K-HATERS, K-MHaS |
| ② 라벨 | 문장과 정답을 짝지음 | 데이터셋별로 다른 포맷을 `(text, label)` 표로 통일 |
| ③ 전처리 | 텍스트 청소 | 특수문자·중복공백 제거, 형태소 분리 등 |
| ④ 벡터화 | 글자를 숫자로 | `CountVectorizer` / `TF-IDF` → [03 문서](03-vectorization.md) |
| ⑤ 학습 | 모델이 패턴 익힘 | 로지스틱 회귀 등 → [04 문서](04-models.md) |
| ⑥ 평가 | 잘하는지 채점 | accuracy·F1 등 → [05 문서](05-evaluation.md) |

## 왜 단계를 나누나요

각 단계의 **결과물을 폴더에 저장**해 두면, 앞 단계를 다시 안 해도 됩니다.
예를 들어 모델만 바꿔 실험할 때 ①~④는 그대로 두고 ⑤만 다시 돌리면 되죠.
그래서 `data/` 폴더가 단계별로 나뉘어 있습니다 → [02 문서](02-data-structure.md).

## 한 가지 주의: 학습과 예측은 입력이 같아야 함

모델은 학습할 때 본 **형식 그대로**의 입력만 이해합니다.
학습 데이터를 TF-IDF로 벡터화했다면, 예측할 새 문장도 **똑같은 TF-IDF 규칙**으로
벡터화해야 합니다. 그래서 벡터화 도구(vectorizer)도 모델과 함께 저장해 둡니다.
이걸 잊으면 "학습은 잘됐는데 실제 채팅에선 엉뚱한 결과"가 나옵니다.
