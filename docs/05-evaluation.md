# 5. 평가 지표

모델을 학습시켰으면 **"얼마나 잘하나"**를 숫자로 재야 합니다.
그래야 여러 후보 모델 중 무엇을 쓸지 정할 수 있습니다.

## 먼저: 학습 데이터와 시험 데이터를 나눈다

학생이 **공부한 문제 그대로** 시험 보면 점수가 의미 없죠.
모델도 마찬가지입니다. 그래서 가진 데이터를 둘로 나눕니다.

| 이름 | 비율 (보통) | 용도 |
| --- | --- | --- |
| 학습 데이터 (train) | 80% | 모델이 패턴을 익히는 데 사용 |
| 시험 데이터 (test) | 20% | 모델이 **처음 보는** 문장으로 채점 |

모델은 test 데이터를 학습 때 **본 적이 없어야** 합니다.
이걸 어기는 실수를 **데이터 누수(data leakage)**라고 하며, 점수가 부풀려져
"학습은 100점인데 실제 채팅에선 엉망"인 사태가 납니다.

> **과적합(overfitting)**: 모델이 학습 데이터만 통째로 외워서, 새 문장에는
> 약해지는 현상. train 점수는 높은데 test 점수가 낮으면 과적합 의심.

## 4가지 결과: 맞춤과 틀림

공격성 탐지를 예로, 모델의 예측은 네 가지로 나뉩니다.

| | 실제 공격적 | 실제 비공격적 |
| --- | --- | --- |
| **공격적이라 예측** | 정답 (TP) | 오탐 (FP) — 멀쩡한 말을 공격적이라 함 |
| **비공격적이라 예측** | 놓침 (FN) — 공격적인 말을 놓침 | 정답 (TN) |

이 네 칸을 모아 그린 표를 **혼동 행렬(confusion matrix)**이라고 합니다.

## 핵심 지표 4개

### 정확도 (Accuracy)

전체 중 맞춘 비율. `(TP + TN) / 전체`.
가장 직관적이지만 **함정**이 있습니다 (아래 "주의" 참고).

### 정밀도 (Precision)

"공격적이라고 한 것 중 **진짜** 공격적인 비율". `TP / (TP + FP)`.
→ 낮으면 **오탐이 많음**. "왜 멀쩡한 말에 경고 떠?"

### 재현율 (Recall)

"실제 공격적인 것 중 **놓치지 않고 잡은** 비율". `TP / (TP + FN)`.
→ 낮으면 **놓침이 많음**. "공격적인 말이 그냥 통과됐어."

### F1 점수

정밀도와 재현율의 **균형 점수**. 둘 다 높아야 F1이 높습니다.
하나만 높고 하나가 낮으면 F1이 깎입니다.
모델 비교에는 보통 **F1을 주 지표로** 씁니다.

## 정밀도 vs 재현율 — ChaeOn은 뭘 중시하나

둘은 보통 **한쪽을 올리면 다른 쪽이 내려가는** 관계입니다.

- **재현율을 중시** → 위험 신호를 놓치지 않음. 대신 오탐(괜한 경고) 증가.
- **정밀도를 중시** → 경고가 뜨면 진짜임. 대신 일부 위험을 놓침.

ChaeOn은 "커뮤니케이션 위험 신호 **조기 탐지**"가 목표라 **재현율이 어느 정도
중요**합니다 (놓치면 의미가 없으니까). 하지만 오탐이 너무 많으면 팀원이 경고를
무시하게 되어 신뢰를 잃습니다. → **F1로 균형을 보면서**, 팀이 "놓침과 오탐 중
무엇이 더 곤란한가"를 정해 기준선을 조정하세요.

## 주의: 정확도만 보면 속는다 (불균형 데이터)

공격성 데이터는 보통 **비공격적 메시지가 훨씬 많습니다.**
예를 들어 100개 중 공격적 메시지가 5개뿐이라면:

> 모델이 무조건 "전부 비공격적"이라고만 찍어도 → 정확도 **95%**.
> 하지만 공격적 메시지 5개를 **전부 놓쳤으니** 재현율은 **0%**. 쓸모없는 모델.

이래서 ChaeOn 같은 탐지 문제는 **정확도만 믿으면 안 되고** 정밀도·재현율·F1을
함께 봐야 합니다. 이런 데이터를 **불균형 데이터(imbalanced data)**라고 합니다.

## ChaeOn에서 평가를 쓰는 흐름

1. 후보 조합을 만든다: (CountVectorizer / TF-IDF) × (로지스틱 회귀 / 랜덤
   포레스트 / 나이브 베이즈).
2. 각 조합을 train으로 학습 → test로 채점.
3. **F1 등 지표를 비교**해 기능 1·2 각각의 최종 모델을 확정.
4. 확정된 모델로 실제 팀 채팅을 예측 → 그 결과를 기능 3·4·5가 집계.

> 평가 지표는 `scikit-learn`의 `classification_report`, `confusion_matrix`로
> 한 번에 뽑을 수 있습니다. 용어가 막히면 [06 용어 사전](06-glossary.md) 참고.
